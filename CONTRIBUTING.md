# 🤝 Guía de contribución

¡Gracias por tu interés en contribuir a este proyecto! Aquí tienes las pautas para hacerlo.

## 📋 Antes de empezar

- Asegúrate de tener instalado **Node.js** (versión 18 o superior) y **Git**.
- Lee el [README.md](README.md) para entender de qué trata el proyecto.
- Revisa los [issues abiertos](../../issues) para ver si alguien ya está trabajando en lo que quieres hacer.

## 🚀 ¿Cómo contribuir?

### 1. Fork y clonación

```bash
# Haz fork desde la web de GitHub, luego clona tu fork
git clone https://github.com/TU_USUARIO/proyecto-inicial.git
cd proyecto-inicial
```

### 2. Crea una rama

```bash
git checkout -b feature/nombre-descriptivo
```

Usa prefijos claros:
- `feature/` para nuevas funcionalidades
- `fix/` para corregir bugs
- `docs/` para cambios en documentación
- `refactor/` para refactorizaciones

### 3. Haz tus cambios

- Escribe código limpio y comentado cuando sea necesario.
- Sigue el estilo de código del proyecto.
- Añade pruebas si aplica.

### 4. Commits descriptivos

Usa el formato [Conventional Commits](https://www.conventionalcommits.org/es/):

```
feat: agregar formulario de login
fix: corregir validación de email
docs: actualizar instrucciones de instalación
```

### 5. Push y Pull Request

```bash
git push origin feature/nombre-descriptivo
```

Luego abre un **Pull Request** describiendo:
- ¿Qué cambió?
- ¿Por qué?
- ¿Cómo se prueba?

## 🐛 Reportar bugs

Abre un [issue](../../issues/new) con:
- Descripción clara del problema
- Pasos para reproducirlo
- Comportamiento esperado vs. real
- Capturas de pantalla si aplica

## 💡 Proponer mejoras

Las ideas son bienvenidas. Abre un issue describiendo:
- El problema que resuelve
- La solución propuesta
- Alternativas consideradas

## 📜 Código de conducta

Sé respetuoso y constructivo. Tratamos a todos con cortesía.

---

¡Gracias por contribuir! 🎉
